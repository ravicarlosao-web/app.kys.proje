VISÃO GERAL DO PROJETO
Cria um aplicativo mobile completo chamado "MediLembrete" usando React Native com Expo. O app deve funcionar em Android e iOS, ser totalmente funcional offline, ter visual profissional de nível internacional, e permitir que o utilizador registe medicamentos com horários e receba notificações locais de lembrete.

STACK TECNOLÓGICA OBRIGATÓRIA

Framework: React Native com Expo (SDK mais recente)
Linguagem: JavaScript (ou TypeScript se possível)
Navegação: React Navigation v6 (Stack + Bottom Tab Navigator)
Armazenamento local: AsyncStorage (@react-native-async-storage/async-storage)
Notificações: expo-notifications
Ícones: @expo/vector-icons (família Ionicons)
Data e hora: date-fns (para formatação de datas)
UUID: uuid ou nanoid (para gerar IDs únicos dos medicamentos)
Fontes: expo-font com Google Fonts (usar fonte Poppins)

FUNCIONALIDADES COMPLETAS
1. ECRÃ INICIAL — HomeScreen
O que deve mostrar:

Header com logo pequeno à esquerda e título "MediLembrete" ao centro
Subtítulo: "Os teus medicamentos de hoje"
Data de hoje formatada em português (ex: "Sexta, 30 de Maio de 2026")
Resumo do dia em 3 cards horizontais no topo:

Card 1: Total de medicamentos cadastrados
Card 2: Doses tomadas hoje
Card 3: Doses pendentes hoje


Lista de medicamentos cadastrados, cada um como um card com:

Ícone da categoria do medicamento (comprimido, xarope, injeção, etc.)
Nome do medicamento em negrito
Dosagem (ex: "500mg")
Próximo horário de toma
Indicador colorido de status (tomado/pendente)
Botão de marcar como "tomado" diretamente no card


Botão flutuante (FAB) no canto inferior direito com ícone "+" para adicionar novo medicamento
Estado vazio: se não houver medicamentos, mostrar ilustração com texto "Nenhum medicamento cadastrado. Toca no + para adicionar."

Interações:

Toque no card → navega para DetalhesScreen
Toque no FAB → navega para AddMedicamentoScreen
Toque no botão "tomado" → marca dose como tomada e atualiza o estado visualmente com animação


2. ECRÃ ADICIONAR MEDICAMENTO — AddMedicamentoScreen
Campos do formulário:

Nome do medicamento (obrigatório)

TextInput com placeholder "Ex: Paracetamol"
Validação: não pode estar vazio


Dosagem (obrigatório)

TextInput com placeholder "Ex: 500mg, 10ml, 1 comprimido"


Categoria (obrigatório)

Seleção visual em grid com ícones:

💊 Comprimido
🧴 Xarope
💉 Injeção
🩹 Pomada
👁️ Colírio
💨 Inalador


O item selecionado fica com fundo verde e borda destacada


Cor do medicamento (opcional)

Paleta de 8 cores para identificar visualmente o medicamento na lista
Cores: verde, azul, vermelho, laranja, roxo, rosa, amarelo, cinza


Data de início (obrigatório)

DatePicker nativo do Expo (expo-datetime-picker)
Default: hoje


Data de fim / Duração (opcional)

Toggle switch: "Tem data de fim?"
Se ativado, mostrar DatePicker para data de fim


Horários de toma (obrigatório — mínimo 1)

Botão "Adicionar horário" que abre TimePicker
Lista dos horários já adicionados com botão de remover cada um
Pode adicionar até 6 horários por dia
Cada horário aparece como uma tag/chip removível


Dias da semana (obrigatório)

Seleção múltipla dos dias: Seg / Ter / Qua / Qui / Sex / Sáb / Dom
Botão "Todos os dias" para selecionar todos de uma vez
Dias selecionados ficam com fundo verde


Instruções / Observações (opcional)

TextInput multilinha
Placeholder: "Ex: Tomar com água, após as refeições"


Estoque (opcional)

Campo numérico: quantidade atual de comprimidos/doses disponíveis
Alertar quando estoque estiver baixo (menos de 5 unidades)



Botões:

"Guardar Medicamento" — botão verde grande na parte inferior
"Cancelar" — botão secundário acima do guardar

Validações:

Se nome ou dosagem estiverem vazios → mostrar mensagem de erro abaixo do campo
Se nenhum horário for adicionado → mostrar erro
Se nenhum dia da semana for selecionado → mostrar erro

Ao guardar:

Gerar ID único com uuid/nanoid
Salvar no AsyncStorage
Agendar notificações locais para cada horário nos dias selecionados
Navegar de volta para HomeScreen com mensagem de sucesso (Toast ou Snackbar)


3. ECRÃ EDITAR MEDICAMENTO — EditMedicamentoScreen

Idêntico ao AddMedicamentoScreen mas com os campos pré-preenchidos
Recebe o ID do medicamento via params da navegação
Ao guardar: atualiza o AsyncStorage, cancela as notificações antigas e reagenda as novas
Botão vermelho "Eliminar Medicamento" no final do formulário com modal de confirmação


4. ECRÃ DETALHES — DetalhesScreen
O que mostrar:

Header com nome do medicamento e botão de editar (ícone lápis) no topo direito
Card grande com:

Ícone da categoria
Nome e dosagem em destaque
Cor identificadora
Data de início e fim
Dias da tomada
Instruções
Nível de estoque com barra de progresso visual


Secção "Horários de Hoje":

Lista de todos os horários do dia
Cada horário com status: ✅ Tomado / ⏰ Pendente / ❌ Saltado
Botão "Marcar como tomado" para os pendentes


Secção "Histórico Recente":

Últimos 7 dias com indicadores coloridos (tomado/saltado/pendente)




5. ECRÃ HISTÓRICO — HistoricoScreen
O que mostrar:

Filtro por medicamento (dropdown/picker)
Filtro por período: Hoje / Esta semana / Este mês
Lista de registos com:

Data e hora
Nome do medicamento
Status (tomado/saltado)
Ícone colorido por status


Resumo no topo: percentagem de adesão ao tratamento (ex: "Aderência: 85%")

ANIMAÇÕES E MICRO-INTERAÇÕES

Marcar como tomado: botão faz scale de 1.0 → 0.9 → 1.0 com mudança de cor
Adicionar medicamento: card novo entra com fadeIn + slideInUp
Eliminar medicamento: card sai com fadeOut + slideOutRight
FAB: rotação de 0° para 45° ao abrir (se tiver menu), pulse suave
Loading states: ActivityIndicator verde nas operações assíncronas
Toast de sucesso: aparece no topo por 3 segundos e desaparece com fadeOut


TRATAMENTO DE ERROS E EDGE CASES

Se o utilizador negar permissão de notificações → mostrar alerta explicando a importância e botão para ir às definições
Se o AsyncStorage falhar → mostrar mensagem de erro e não fechar o ecrã
Se não houver medicamentos → mostrar EmptyState
Medicamentos com data de fim ultrapassada → mostrar como "Concluído" com cor diferente
Estoque em zero → mostrar alerta vermelho "Estoque esgotado" no card

INSTRUÇÕES FINAIS PARA O REPLIT

Cria o projeto com: npx create-expo-app MediLembrete
Instala todas as dependências listadas acima
Implementa cada ficheiro seguindo exatamente a estrutura de pastas definida
Usa a paleta de cores definida em src/constants/colors.js
Carrega a fonte Poppins com expo-font antes de renderizar qualquer ecrã
O app deve funcionar 100% offline — nenhuma chamada a API externa
Todas as notificações são locais (expo-notifications)
Testa o fluxo completo: adicionar → ver na lista → receber notificação → marcar como tomado → ver no histórico
O código deve estar comentado em português
Usa StyleSheet.create() para todos os estilos, nunca estilos inline
